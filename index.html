Light-weight prototyping with HTML & SCSS

Livewires is simple HTML & SCSS. It’s not ugly, fancy. It doesn’t look like a final design, but it also doesn’t look like dog doo. Most text is represented by background images. It is designed to be a starting point for production code, not something you throw out. It's also very fast to work with. It is more of a philosophy than a product.

Fork it on Github

Try Livewires if you…

  • Need to quickly protoype responsive layouts

  • Find most frameworks & libraries over-designed & bloated

  • Love working in HTML & SCSS

  • Are sick of paying for ugly buggy wireframing apps

  • Want something you can easily test on multiple devices

  • Hate wasting time on throw-away deliverables

How to use

  1. Drop a copy of Livewires into the location of your new website project
  2. Point your preferred SCSS compiler to Livewires (I recommend Codekit)
  3. Create HTML files for each of your templates in httpdocs
  4. Write your structural HTML using the recommended classes
  5. Layout your templates with SCSS in /scss/custom

Recommended classes

.content

Elements with the content class will load the text placeholder as a repeating background image. The height of the element determines how many lines of “text” appear.

Example:
.content { height: $line*5; }

.link-list

List items load a single-line text placeholder by default. The link-list class removes the image when placed on the containing list element.

.callout

The callout class reverses the colours of the element it’s applied to. Useful for side bars or other highlighted content like the box to the right.

Tips

  • Adjust the height of .content elements using the $line variable (e.g. height:$line*5; will render 5 lines of "text").
  • See how placeholder images are configured at Placehold.It.
  • /scss/livewires/readymade/ has starting points for commonly used UI patterns.
  • Keep your usable structural SCSS separate from the default Livewires styles and .content height adjustments.
  • Reference actual content to approximate content blocks. Even though we’re hiding text, it should be based in reality.
  • Throw away as little of your work as possible. A Livewires prototype should be an evolution, not another discarded deliverable.
