Light-weight prototyping with HTML & SCSS

Live Wires is a simple library for creating visually-neutral, responsive HTML prototypes. Forget about ugly wireframing apps and bloated UI libraries. Start evolving your projects with your own markup and a little generic style help from Live Wires.

Read about how we use Live Wires on the nGen Works blog.

Fork it on Github

Try Live Wires if you…

  • Need to prototype responsive layouts quickly

  • Find most frameworks & libraries over-designed & bloated

  • Love working in HTML & SCSS

  • Are sick of paying for ugly, buggy wireframing apps

  • Want easy testing on multiple devices

  • Hate wasting time on throw-away deliverables

How to use

  1. Drop a copy of Live Wires into the location of your new website project.
  2. Point your preferred SCSS compiler to Live Wires (I recommend Codekit).
  3. Create HTML files for each of your templates in httpdocs.
  4. Write your structural HTML using the recommended classes instead of content or lorem ipsum.
  5. Layout your templates with SCSS in /scss/custom.

Recommended classes

.content

Elements with the content class will load the text placeholder as a repeating background image. The height of the element determines how many lines of “text” appear.

Example:
.content { height: $line*5; }

.link-list

List items load a single-line text placeholder by default. The link-list class removes the image when placed on the containing list element.

.callout

The callout class reverses the colours of the element to which it is applied. Useful to denote sidebars or other important content like the Tips callout shown.

Tips

  • Adjust the height of .content elements using the $line variable (e.g. height:$line*5; will render 5 lines of "text").
  • See how placeholder images are configured at Placehold.It.
  • /scss/livewires/readymade/ has starting points for commonly used UI patterns.
  • Reference actual content to approximate content blocks. Even though we’re hiding text, it should be based in reality.
  • Live Wires is not a design library. You don’t want to use the default styles for your app’s actual UI.
  • Keep your usable structural SCSS separate from the default Live Wires styles and .content height adjustments. A Live Wires prototype should be an evolution, not another discarded deliverable.
