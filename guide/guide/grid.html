Grid

Live Wires uses a folding compound grid for maximum flexibility in responsive layouts. This is a fluid grid with flexible columns and fixed-width gutters to maintain proportional spacing and alignment for all screen dimensions.

Avoiding a grid framework

The concept is loosely based on Joni Korpi’s fabulous Golden Grid System.

Unlike other heavy grid frameworks, the grid system is not attached to CSS classes. Instead, they are set in SCSS using a simple grid variables. This keeps the markup clean and the layout much easier to modify at various breakpoints. You can even combine them to make interesting compound grids.

In fact, this system isn't a framework at all. The only thing we've setup for you are the following grid variables and a couple classes for dividing page sections.

Default grid variables

   //Three Column Grid
    $c3: 100% / 3;
    //Four Column Grid
    $c4: 100% / 4;
    // Five Columns
    $c5: 100% / 5;
    //Six Column Grid
    $c6: 100% / 6;
    //Seven Column Grid
    $c7: 100% / 7;
    //Eight Column Grid
    $c8: 100% / 8;
    //Nine Column Grid
    $c9: 100% / 9;
    //Ten Column Grid
    $c10: 100% / 10;
    //Twelve Column Grid
    $c12: 100% / 12;
    //Sixteen Column Grid
    $c16: 100% / 16;

To specify a width of four columns of a six column grid ($c6 == 1 column of a 6-column grid), use the following:

    /* Create a 4-column object centered in a 6-column grid */
    .object{
      @include gutters; /* Applies gutter spacing to sides of object. */
      width:$c6*4; /* $c6 = 1 column of a 6-column grid. Mulitply by 4 to specify 4 columns wide. */
      margin-left:$c6*1; /* Moves object 1 column to the right. */
    }

By default, this grid would be from one edge of the browser to the other. Probably not ideal for most people. To give your page some margins, we have two classes to give you some structure and extra theming power:

    <!-- If you don't need a unique background for your section,
    you can combine row and container classes -->
    <div class="row container">
      <section class="object">
        <!-- Content that aligns to the grid -->
      </section>
    </div>
    
    <div class="row"> <!-- Apply background here -->
      <div class="container"> <!-- Contains grid -->
        <section class="object">
          <!-- Content that aligns to the grid -->
        </section>
        <section class="object2">
          <!-- Content that aligns to the grid -->
        </section>
      </div>
    </div>

.row

Our row classes are designed to stretch from one edge of your browser to the other so you can add background colors and full-bleed images. We also like to add some vertical padding for consistent white space between sections.

We’ve included a few background classes in /source/scss/theme/_layout.scss to give you some easy background variations for rows and other elements:

.subtle

This content has a subtle background variation.

    <section class="row subtle">
    <div class="container">
      <h3>.subtle</h3>
      <p>This content has a subtle background variation.</p>
    </div>
  </section>

.reverse

This content has light on dark text.

    <section class="row reverse">
    <div class="container">
      <h3>.reverse</h3>
      <p>This content has light on dark text.</p>
    </div>
  </section>

.container

Container classes contain our grid and add vertical margins to our sections. You can add a max-width value to give your design a static width after a certain size, but we recommend sticking with a fully fluid system based on percentages:

Define your page width by using the $margin variable. We usually set this with a percentage based on one of our smaller grid units. For example, the base margin of Live Wires is $margin: 100% / 18 (which adds an extra column to either side of our 16 column grid).

This is fine at small window sizes, but what about at wider sizes? Instead of adding a max-width, we just increase the $margin variable on our .container classes:

    /* Increase margins after a certain width */
    @media only screen and (min-width: $screenMedium) {
      .container {
        margin: 0 $margin * 2; /* Double the margin size to squeeze the main content. */
      }
    }

Units & resizing

Every single dimension in our project is defined using relative units. Aside from our grid units and margins, which use percentages, we use Rems. When everything is set using rems, we can proportionally zoom the entire project simply by changing the font-size of our <html> element.

Example

   // Our default font-size is 100%
    html {
      font-size: 100%;
    }

    // We can proportionally increase the size of the enire site
    // by simply increasing or decreasing the root font-size.
    @media only screen and (min-width: $screenSmall) {
      html {
        font-size: 105%; // Increases everything defined by rems by 5%.
      }
    }

To make it easy to work with rems, we've included a handy rem() function inspired by Foundation’s rem-calc().

Examples

    .object {
       // Here's a few ways you can use the rem() function
       padding: rem(24px); // Converts a px value to rems
       padding: rem(24); // You can even leave out the px suffix
       padding: rem(24 12); // And use with multiple shorthand values
       padding: rem($line $line/2); // Use the $line variable for consistent spacing & flexibility
    }
